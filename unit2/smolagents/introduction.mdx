# `smolagents` 소개

<img src="https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit2/smolagents/thumbnail.jpg" alt="Unit 2.1 Thumbnail"/>

이 모듈에서는 [`smolagents`](https://github.com/huggingface/smolagents) 라이브러리를 사용하여 **효과적인 에이전트를 구축하는 방법**을 배우게 됩니다. 이 라이브러리는 유능한 AI 에이전트를 만들기 위한 경량 프레임워크를 제공합니다.

`smolagents`는 Hugging Face 라이브러리입니다. smolagents [`리포지토리`](https://github.com/huggingface/smolagents)에 **스타**를 눌러 지원해주시면 감사하겠습니다:

## 모듈 개요

이 모듈은 `smolagents`를 사용하여 지능형 에이전트를 구축하기 위한 주요 개념과 실용적인 전략에 대한 포괄적인 개요를 제공합니다.

많은 오픈소스 프레임워크가 있는 상황에서, `smolagents`를 유용한 옵션으로 만드는 구성 요소와 기능을 이해하거나, 다른 솔루션이 더 적합할 수 있는 경우를 판단하는 것이 중요합니다.

우리는 소프트웨어 개발 작업을 위한 코드 에이전트, 모듈식 함수 기반 워크플로우를 만들기 위한 도구 호출 에이전트, 정보에 접근하고 종합하는 검색 에이전트 등 중요한 에이전트 유형들을 살펴볼 것입니다.

또한 여러 에이전트의 조율과 함께 비전 기능과 웹 브라우징의 통합도 다룰 것입니다. 이를 통해 동적이고 상황을 인식하는 애플리케이션을 위한 새로운 가능성이 열립니다.

이 유닛에서는 Unit 1의 에이전트인 Alfred가 돌아옵니다. 이번에는 내부 작동 방식으로 `smolagents` 프레임워크를 사용합니다. 우리는 함께 Alfred가 다양한 작업을 수행하면서 이 프레임워크의 주요 개념들을 탐험할 것입니다. Alfred는 Wayne 가족 🦇이 없는 동안 Wayne Manor에서 파티를 준비하고 있으며, 할 일이 많습니다. `smolagents`를 사용하여 이러한 작업들을 어떻게 처리하는지 그의 여정을 함께 살펴보겠습니다!

<Tip>

이 유닛에서는 `smolagents` 라이브러리로 AI 에이전트를 구축하는 방법을 배웁니다. 여러분의 에이전트는 데이터를 검색하고, 코드를 실행하고, 웹 페이지와 상호작용할 수 있게 될 것입니다. 또한 여러 에이전트를 결합하여 더 강력한 시스템을 만드는 방법도 배우게 됩니다.

</Tip>

## 목차

이 `smolagents` 유닛에서는 다음 내용을 다룹니다:

### 1️⃣ [smolagents를 사용하는 이유](./why_use_smolagents)

`smolagents`는 애플리케이션 개발에 사용할 수 있는 많은 오픈소스 에이전트 프레임워크 중 하나입니다. 대안으로는 이 코스의 다른 모듈에서 다루는 `LlamaIndex`와 `LangGraph`가 있습니다. `smolagents`는 특정 사용 사례에 매우 적합할 수 있는 여러 주요 기능을 제공하지만, 프레임워크를 선택할 때는 항상 모든 옵션을 고려해야 합니다. 우리는 `smolagents` 사용의 장단점을 살펴보면서, 여러분의 프로젝트 요구사항에 기반한 정보에 입각한 결정을 내리는 데 도움을 줄 것입니다.

### 2️⃣ [CodeAgents](./code_agents)

`CodeAgents`는 `smolagents`의 주요 에이전트 유형입니다. JSON이나 텍스트를 생성하는 대신, 이 에이전트들은 작업을 수행하기 위한 Python 코드를 생성합니다. 이 모듈에서는 그들의 목적, 기능, 작동 방식을 살펴보고, 실습 예제를 통해 그들의 능력을 보여줍니다.

### 3️⃣ [ToolCallingAgents](./tool_calling_agents)

`ToolCallingAgents`는 `smolagents`가 지원하는 두 번째 유형의 에이전트입니다. Python 코드를 생성하는 `CodeAgents`와 달리, 이 에이전트들은 시스템이 작업을 실행하기 위해 파싱하고 해석해야 하는 JSON/텍스트 블롭에 의존합니다. 이 모듈에서는 그들의 기능, `CodeAgents`와의 주요 차이점을 다루고, 예제를 통해 사용법을 설명합니다.

### 4️⃣ [Tools](./tools)

Unit 1에서 보았듯이, 도구는 LLM이 에이전트 시스템 내에서 사용할 수 있는 함수이며, 에이전트 행동의 필수적인 구성 요소입니다. 이 모듈에서는 `Tool` 클래스나 `@tool` 데코레이터를 사용하여 도구를 만드는 방법, 그들의 구조, 다양한 구현 방법을 다룹니다. 또한 기본 도구상자, 커뮤니티와 도구를 공유하는 방법, 여러분의 에이전트에서 사용하기 위해 커뮤니티가 기여한 도구를 로드하는 방법도 배우게 됩니다.

### 5️⃣ [Retrieval Agents](./retrieval_agents)

검색 에이전트는 모델이 지식 베이스에 접근할 수 있게 하여, 여러 소스에서 정보를 검색하고, 종합하고, 가져올 수 있게 합니다. 이들은 효율적인 검색을 위해 벡터 저장소를 활용하고 **검색 증강 생성(RAG)** 패턴을 구현합니다. 이 에이전트들은 특히 웹 검색을 커스텀 지식 베이스와 통합하면서 메모리 시스템을 통해 대화 컨텍스트를 유지하는 데 유용합니다. 이 모듈에서는 강력한 정보 검색을 위한 폴백 메커니즘을 포함한 구현 전략을 살펴봅니다.

### 6️⃣ [Multi-Agent Systems](./multi_agent_systems)

여러 에이전트를 효과적으로 조율하는 것은 강력한 다중 에이전트 시스템을 구축하는 데 중요합니다. 웹 검색 에이전트와 코드 실행 에이전트와 같이 서로 다른 능력을 가진 에이전트들을 결합함으로써, 더 정교한 솔루션을 만들 수 있습니다. 이 모듈은 효율성과 신뢰성을 극대화하기 위해 다중 에이전트 시스템을 설계, 구현, 관리하는 데 중점을 둡니다.

### 7️⃣ [Vision and Browser agents](./vision_agents)

비전 에이전트는 **비전-언어 모델(VLM)**을 통합하여 시각적 정보를 처리하고 해석할 수 있도록 전통적인 에이전트 기능을 확장합니다. 이 모듈에서는 이미지 기반 추론, 시각적 데이터 분석, 멀티모달 상호작용과 같은 고급 기능을 활용할 수 있도록 VLM 기반 에이전트를 설계하고 통합하는 방법을 살펴봅니다. 또한 비전 에이전트를 사용하여 웹을 탐색하고 정보를 추출할 수 있는 브라우저 에이전트를 구축할 것입니다.
